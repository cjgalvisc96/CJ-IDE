# CJ-IDE tool manifest — the SINGLE SOURCE OF TRUTH for what mise installs.

# Both install.sh and prune.sh read this file, so there is exactly one list to
# maintain. Format:
#   * one `mise` tool spec per line  (e.g. node@lts, go:host/pkg@latest)
#   * blank lines and lines starting with `#` are ignored
#   * `# [section]` headers group tools; install.sh installs runtimes first
#     (so the go:/npm: backends below can build)
#
# VERSIONS ARE PINNED TO major.minor for reproducibility: a fresh install on a
# new machine resolves to the SAME tools, while still picking up patch releases
# (mise reads `ruff@0.15` as "latest 0.15.x"). To move to a newer feature
# release, bump the number here deliberately and re-run ./install.sh — never let
# it float. (An unpinned Neovim jump is what broke treesitter once already.)
